## 1. Objective

Take an existing pretrained ~1.5B-parameter coding model and fine-tune it on
instruction-style coding data using QLoRA, producing a more capable coding model
that runs locally on a 16GB no-GPU laptop and will power a CLI coding agent
(Project 3).

The core hypothesis: a small base coding model, when instruction-tuned on
high-quality coding Q&A data, becomes meaningfully better at following coding
instructions — measurable on a standard benchmark.

---

## 2. Approach

### 2.1 Base model
**Qwen2.5-Coder-1.5B** (the *base/pretrained* version, not Instruct).
- 1.5 billion parameters, ~3 GB in fp16.
- Apache 2.0 license (verified) — permits fine-tuning and portfolio/commercial use.
- Chosen because it's the strongest open small coder in its class (~41% HumanEval
  per the paper). We use the *base* version because the whole project IS the
  instruction-tuning step — using the Instruct version would defeat the purpose.

### 2.2 Dataset
**Magicoder-OSS-Instruct-75K** (20,000-example slice).
- 75,197 synthetic instruction-coding examples generated by feeding real
  open-source code snippets to GPT-4.
- Fields: `problem` (the instruction), `solution` (the code answer), `lang`,
  `seed` (the OSS snippet that inspired it).
- We use 20k examples for the first run (enough to see improvement, small enough
  to train in ~2.5 hours on a free GPU).

### 2.3 Method: QLoRA
**Q**uantized **Lo**w-**R**ank **A**daptation. Two ideas combined:

1. **LoRA:** Instead of updating all 1.5B parameters during fine-tuning, we bolt
   on tiny "adapter" matrices (~0.5% of the model) and only train those. The
   original weights stay frozen. This makes training cheap.

2. **Quantization (the "Q"):** The frozen original model is shrunk from 16-bit to
   4-bit numbers (4× smaller), so a 1.5B model fits in ~6 GB of GPU memory
   instead of ~12 GB. This lets us train on a free GPU.

The trained adapter is then **merged** back into the base model, producing one
complete fine-tuned model.

### 2.4 Tooling
- **Unsloth** for the QLoRA training (2× faster, ~40% less memory than vanilla
  HuggingFace, with first-class Qwen2.5 + GGUF support).
- **HuggingFace transformers/datasets** for model/dataset loading.
- **llama.cpp** (via Unsloth) for GGUF export.
- **Ollama** for local inference testing.

### 2.5 Platform
- **Kaggle free tier** (Tesla T4, 16 GB VRAM) — sufficient for 1.5B QLoRA (~6 GB
  needed). Total GPU cost: **₹0**.
- Fallback: Vast.ai RTX 3060 (~₹170 total) if Kaggle quota is exhausted.

### 2.6 Hyperparameters (conservative first run)
| Parameter | Value | Rationale |
|---|---|---|
| LoRA rank | 16 | Standard default; balances capacity and overfitting risk |
| LoRA alpha | 32 | Conventionally 2× rank |
| Target modules | all 7 linear layers | Maximize adapter expressiveness |
| Epochs | 3 | Conservative; small model overfits easily |
| Learning rate | 2e-4 | Standard QLoRA rate |
| Batch size | 2 × 4 (grad accum) = 8 effective | Memory-safe on T4 |
| Max seq len | 2048 | Covers 100% of sampled examples (verified) |
| Save steps | 200 | Checkpoint often; dropped session loses ≤10 min |

## 3. Benchmark: HumanEval pass@1

### 3.1 What it measures
HumanEval is OpenAI's standard coding benchmark: 164 hand-written Python problems,
each with a function signature + docstring and official test cases. **pass@1** =
fraction of problems where the model's generated code passes the tests.

### 3.2 Evaluation methodology
- **Base model:** tested in `completion` mode (raw prompt → code continuation).
  Matches the Qwen2.5-Coder paper's methodology for comparability.
- **Fine-tuned model:** tested in `instruction` mode (chat-wrapped prompt → code
  block). Matches how the model will be used in Project 3.
- **Greedy pass@1** (n=1, temperature=0): deterministic, reproducible, fast.
- Generated code executed in a sandboxed subprocess with a 10-second timeout.

## 6. Honest limitations

This is a **~1.5B parameter model.** It is:

**Good at:**
- Writing small, well-specified functions ("write a function to reverse a string")
- Explaining short code snippets
- Simple bug fixes in single functions
- Completing common patterns (fizzbuzz, palindromes, sorting)

**Poor at:**
- Multi-file reasoning or understanding large codebases
- Complex algorithms (dynamic programming, graph algorithms beyond basics)
- Long contexts (>2048 tokens)
- Following many-step instructions
- Novel or unusual problems not represented in training data

**Not a replacement for:** Claude, GPT-4, or any frontier model. This project
demonstrates the *method* (QLoRA fine-tuning with honest before/after
benchmarking) on a small, cheap-to-run model — not frontier capability.

### Methodology caveats
- Base and fine-tuned models are benchmarked in *different modes* (completion vs
  instruction), each in the mode it was designed for. This is defensible but not
  purely apples-to-apples. See `src/benchmark.py` for the full rationale.
- Greedy pass@1 (n=1) is deterministic but slightly optimistic vs the paper's
  n=20 sampling-based pass@1. The `--n-samples 20` flag is available for a more
  rigorous measurement.
- HumanEval is one benchmark. Real-world coding ability is broader than 164
  small problems can measure.
- Training on 20k examples for 3 epochs teaches the assistant format and nudges
  coding ability; it does not add fundamentally new knowledge.

## 8. Reproducibility

All code, configs, and hyperparameters are in this repository. To reproduce:

1. **Validate locally (free):**
   ```bash
   pip install -r requirements-stage1.txt
   python scripts/validate_format.py
   python scripts/smoke_test_cpu.py
   ```
2. **Run training + export (Kaggle free T4):** follow `scripts/kaggle_run.md`.
3. **Benchmark (Kaggle free T4):** run `scripts/run_benchmark.py` for base and
   fine-tuned models.
4. **Test locally:** follow `scripts/local_gguf_test.md`.

Random seed: 42 (set in training and LoRA init). Greedy benchmark decoding is
fully deterministic.

The fine-tuned GGUF model will power a CLI coding agent (Project 3). The agent

cloud-based frontier model. It excels at scoped tasks and should defer on
